# v0.4.5 — Missing Reference Features Gap Analysis

Audience: planning reference for choosing the next feature batch.

The engine implements most original-reference subsystems to a working level. This
document catalogues the features that are **not yet built** (as opposed to
provisional/placeholder), ordered by estimated impact.

---

## A. Not implemented (no code exists)

| Feature | Reference analogue | Impact | Blocked by |
|---------|--------------|--------|------------|
| **Tech trading** | Trade techs between players via diplomacy screen | Major — core diplomatic mechanic | No trade-negotiation UI, no `Commands.tech_trade` |
| **Resource trading** | Trade strategic/luxury resources per-turn or per-game | Major — eliminates negotiation depth | No trade-negotiation UI, no per-resource trade state |
| **Map trading** | Exchange world-reveal maps | Minor — tactical/diplomatic | No trade-negotiation UI |
| **Open Borders** | Agreement allowing military/civilian passage through territory | Moderate — affects military access, trade routes, religion spread | No agreement lifecycle; `Alliance` has no treaty subtypes |
| **Defensive Pacts** | Mutual military assistance pact | Moderate — reduces diplomatic options | No pact lifecycle in `Alliance` |
| **Peace treaties** | Enforced minimum duration before re-declaring war | Moderate — peace is immediately breakable | No treaty-duration field or enforcement hook |
| **Trade negotiation UI** | "What will you give me for X?" table | Major — the diplomacy screen is war/peace only | Requires the full trade model |
| **Colonies** | Liberate a city as a new independent civ | Minor | `Alliance` has subordination but no colony-spawn path |
| **Simultaneous turns (MP)** | Players act in parallel, resolve at end of turn | Moderate — documented as "planned, not built" | Turn pipeline is strictly sequential |

## B. Event system (stub)

The original reference defines ~100 random events (quests, natural disasters, goody-hut
outcomes, civic/resource-triggered choices). This engine has:

| Item | Status |
|------|--------|
| Event framework (`events.gd`) | Implemented — draws from `data/events.json`, dispatches via `_trigger_event` |
| Events defined | **1** — `ancient_windfall` (gold from an ancient cache) |
| Exploration rewards | Partially — goody huts award gold via the same event path |
| Player-triggered events | None — no quests, no civic/resource/discovery hooks |
| Event choices | None — no branching player-choice popup |

## C. AI espionage

`PlayerAI` (`src/api/player_ai.gd`) drives a competent deterministic turn
across economy, production, units, and research. One gap:

- **AI never touches the intel slider** (`slider_intel` stays 0).
- **AI never launches espionage missions** (`ESPIONAGE_MISSION` command is
  human-only).
- The `IDs.Phase.PLAYER_BOOKKEEPING` phase remains a `pass` — it was
  reserved for an AI planning/intel allocation subsystem.

---

## D. Assembly resolution effects not enforced

Two of 9 resolution effects from `data/resolutions.json` are "recorded only"
— the resolution passes and is stored in `gs.assembly.passed_resolutions`
but has no mechanical effect:

| Resolution | Effect | Status |
|------------|--------|--------|
| Resolution of Open Worship | Free belief spread among member civs | Recorded only |
| Non-Proliferation | Forbid nuclear units for member civs | Recorded only |

The defiance‑of‑rulings contentment penalty (§4.5) is also unwired — a civ
that defies an assembly resolution should suffer empire‑wide anger, but no
contentment effect exists.

## E. Fully implemented but still provisional

The following are coded, wired, and pass tests, but carry `⚠️ Provisional`
markers because their numeric constants are rough first guesses (ported from
the original reference or ballparked) and have not been balance-verified in full playthroughs:

| Area | Constants to tune |
|------|------------------|
| Culture revolt / city flipping (§4.9) | Revolt power thresholds, garrison check weights, belief multiplier |
| Nuclear weapons (§5.7) | Blast radius, unit/city damage %, fallout chance, interception chance |
| Wild forces spawning (§9.2) | Per-difficulty spawn rates, wave sizes, camp cooldowns |
| Wild animals (§9.3) | Spawn frequency, strength scaling, withdraw chance |
| Naval raiders (§9.4) | Patrol radii, target selection, no goal-seeking |
| Era growth scaling (§2.1) | Population growth thresholds per era, revolt-era term |
| Trade route yields (§6.7) | Base yield, distance multiplier, foreign bonus |
| Espionage accumulation (§7.1) | Per-turn EP formula, cost-advantage cap, interception rate |
| Missionary spread (§8.2) | Base spread chance per belief, pressure modifiers |
| State religion (§8.1) | Anarchy turns on switch |
| Difficulty handicaps (§2.2) | Per-level AI bonus %, growth/health/happiness modifiers |
| Tile maturation (§4.10) | Cottage→hamlet→village→town upgrade turns |
| Feature clearing/chopping (§4.11) | Base yield, tech bonus multiplier, border scaling |
| Naval blockade (§5.8) | Commerce penalty range/duration |
| Conscription/draft (§6.6) | Unit cost in population, anger duration |
| Diplomatic victory thresholds (§7.3) | UN 60%, AP 75%, eligibility gates, "too-big" rule |

## F. Blocked on missing subsystems (civic effects inert)

Three policy effects exist in `data/policies.json` but have no game effect
because the subsystem they modify is absent:

| Civic | Effect | Blocked by |
|-------|--------|-----------|
| Caste System | `unlimited_specialists` | No per-building specialist-slot ceiling to lift |
| Free Market | `corporation_maintenance_reduction` | Corps charge per-spread cost, not ongoing maintenance |
| Emancipation | Cross-faction unhappiness | No inter-player happiness model |
